	Firebase
		** Remove the record counts for the modules
		** Create the following admin panel ***
		## **1. Connection Status**
		Shows whether the client is connected to Firestore.
		* **Connected / Syncing / Offline**
		* Last successful sync timestamp
		* Pending writes count
		Useful for diagnosing network issues or offline mode.
		---
		## **2. Firebase Project Info**
		This is non-sensitive and safe because itâ€™s already in the client bundle.
		* **Project ID**
		* **Storage Bucket**
		* **Auth Domain**
		* **App ID**
		* **Measurement ID**
		These help confirm the user is on the correct Firebase project (vs a dev or staging project).
		---
		## **3. Firestore Health & Activity**
		Show:
		* Last read time
		* Last write time
		* Number of successful writes
		* Number of failed writes
		* Firestore latency (ms between request and response)
		This helps debug connection issues.
		---
		## **4. Local Persistence Status**
		Show whether offline persistence is enabled:
		* **IndexedDB active**
		* **Pending local writes**
		* **Cached documents count**
		This helps diagnose data not syncing.
		---
		## **5. Authentication Info**
		Only safe details:
		* User UID
		* Auth provider (email/password, Google, etc)
		* Last login timestamp
		* Is anonymous? yes/no
		Never show tokens, refresh tokens, or email verification links.
		---
		## **6. Realtime Listener Status**
		For apps using `onSnapshot()`:
		* Listener active? yes/no
		* Number of listeners
		* Last update received
		* Errors logged
		Useful for diagnosing realtime update issues.
		---
		## **7. Cloud Storage Connectivity (Optional)**
		Show:
		* Last file upload success
		* Last file download success
		* Storage bucket path
		Useful if your Documents module uploads images or PDFs.
		---
		## **8. Cloud Functions (Optional)**
		If your app uses Functions:
		* Last function call
		* Function name
		* Status (success/error)
		* Latency (ms)
		Helps catch problems such as rate limits or errors.
		---
		## **9. Error Logs**
		Keep a non-sensitive history of:
		* Firestore errors
		* Auth errors
		* Storage upload/download errors
		* Local persistence failures
		* Realtime listener failures
